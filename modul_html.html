Belajar HTML dari Nol

HTML (HyperText Markup Language) adalah bahasa standar untuk membuat halaman web. Di modul ini kita akan belajar tag-tag penting, cara kerja browser membaca kode, dan langsung melihat hasilnya.

01

Heading & Paragraf

HTML menggunakan tag — penanda yang diapit tanda kurung siku — untuk memberi tahu browser cara menampilkan konten. Setiap tag biasanya punya tag pembuka dan tag penutup.


Tag Heading digunakan untuk judul dan sub-judul. Ada 6 tingkatan, dari <h1> (paling besar) sampai <h6> (paling kecil). Tag <p> digunakan untuk paragraf teks biasa.

💡 Analogi
Bayangkan dokumen HTML seperti majalah cetak. <h1> adalah judul utama di halaman depan (paling besar, paling mencolok). <h2> adalah sub-judul artikel. <h3> adalah judul sub-bagian kecil. Sementara <p> adalah isi paragraf biasa yang kamu baca.
TagFungsiUkuran Default
<h1>Judul utama halaman / websitePaling besar (~2em)
<h2>Judul artikel atau sectionBesar (~1.5em)
<h3>Sub-judul / sub-sectionSedang (~1.17em)
<p>Paragraf teksNormal (1em)

<h3>Sub Judul (Heading 3)</h3>

<p>Ini adalah sebuah paragraf.

✅ Tips
Setiap halaman web idealnya hanya punya satu tag <h1> — yaitu judul utama halaman. Ini penting untuk SEO (agar mesin pencari seperti Google mudah memahami halaman kamu).
⚠️ Common Mistake
Jangan pakai heading hanya karena ukurannya besar! Heading punya makna hierarki, bukan sekadar gaya teks. Kalau kamu mau teks besar, gunakan CSS. Heading dipakai untuk struktur konten.

02

Tag Teks Khusus

Selain heading dan paragraf, HTML punya banyak tag untuk memberi makna semantik pada teks — yaitu makna yang bisa dipahami oleh browser, mesin pencari, dan screen reader. Ini berbeda dengan sekadar mengubah tampilan!

💡 Analogi
Bedakan antara format dan makna. Di koran, teks miring bisa berarti judul buku, nama latin hewan, atau kata asing. HTML punya tag berbeda untuk masing-masing maknanya — sehingga mesin bisa mengerti konteksnya, bukan hanya tampilannya.
TagFungsi / MaknaTampilan Default
<cite>Nama sumber / judul karya yang dikutipMiring (italic)
<q>Kutipan singkat inlineDiberi tanda kutip otomatis
<blockquote>Kutipan panjang (blok tersendiri)Menjorok ke dalam
<abbr>Singkatan (dengan tooltip penjelasan)Garis bawah putus-putus
<mark>Teks yang di-highlight / ditandaiBackground kuning
<small>Teks kecil (copyright, catatan kaki)Ukuran lebih kecil

03

List & Nested List

HTML punya tiga jenis list untuk menampilkan data yang berurutan atau berkelompok:

  • Ordered List <ol> — list dengan penomoran (1, 2, 3... atau A, B, C...)
  • Unordered List <ul> — list dengan bullet point (•, ○, ▪)
  • Description List <dl> — list istilah dan definisinya

Setiap item list menggunakan tag <li> (list item), kecuali <dl> yang menggunakan <dt> (term) dan <dd> (description).

💡 Analogi
Ordered List = daftar langkah memasak yang harus urut (1. Cuci beras, 2. Masak air, 3. Masukkan beras...). Unordered List = daftar belanjaan yang urutannya tidak penting. Description List = kamus atau glossary (kata → artinya).
TagAtribut typeContoh tampilan
<ol>type="1" (default), "A", "a", "I", "i"1. 2. 3. / A. B. C. / I. II. III.
<ul>type="disc" (default), "circle", "square"● / ○ / ▪
<dl>—Istilah: definisi

04

Link & Anchor

Tag <a> (anchor) digunakan untuk membuat hyperlink — teks atau gambar yang bisa diklik untuk berpindah ke halaman lain atau ke bagian tertentu dalam halaman yang sama. Atribut wajibnya adalah href yang berisi URL tujuan.

💡 Analogi
Tag <a> seperti jembatan di dunia web. href adalah alamat tujuan jembatan itu mengarah. Kamu bisa membuat jembatan ke halaman lain (eksternal) atau ke bagian lain di halaman yang sama (anchor link).
Jenis LinkSintaks hrefKeterangan
Link eksternalhref="https://google.com"Menuju website lain
Link internalhref="halaman2.html"Menuju file HTML lain
Anchor linkhref="#idnya"Lompat ke elemen dengan id tertentu di halaman sama
Link emailhref="mailto:x@x.com"Membuka aplikasi email
  Demo 4 — Link & Anchor Navigation 
<!-- Menu navigasi anchor -->

<a href="#contact">Contact</a>

<!-- Target anchor (tujuan lompatan) -->

<p>Ini bagian About...</p>

<p>Ini bagian Contact...</p>

<!-- Buka di tab baru -->
<a href="https://google.com"
   target="_blank">
  Buka Google
</a>

05

Media: Gambar, Audio, Video & iFrame

HTML mendukung berbagai jenis media yang bisa langsung ditampilkan di halaman web. Masing-masing menggunakan tag berbeda dengan atribut src (source) yang menunjuk lokasi file.

TagMediaAtribut Penting
<img>Gambar (JPG, PNG, GIF, SVG)src, alt, width, height
<audio>Audio (MP3, WAV, OGG)src, controls, loop, autoplay
<video>Video (MP4, WebM, OGG)src, controls, loop, width
<iframe>Embed halaman lain (YouTube, Maps)src, width, height, allowfullscreen
  Demo 5 — Tag Media 
<!-- Gambar -->
<img src="gambar/foto.jpg"
     alt="Deskripsi gambar"
     width="300px"
     height="200px">

<!-- Audio dengan kontrol -->
<audio src="musik/lagu.mp3"
       controls
       loop></audio>

<!-- Video dengan kontrol -->
<video src="video/film.mp4"
       controls
       loop
       width="400px"></video>

<!-- Embed YouTube via iframe -->
<iframe
  width="400"
  height="225"
  src="https://www.youtube.com/embed/VIDEO_ID"
  allowfullscreen>
</iframe>
⚠️ Selalu pakai atribut alt pada <img>!
Atribut alt wajib diisi — ini adalah teks yang muncul jika gambar gagal dimuat, dan yang dibaca oleh screen reader untuk pengguna tunanetra. Contoh baik: alt="Foto gunung Rinjani saat sunrise". Contoh buruk: alt="gambar" atau tidak ada alt sama sekali.
ℹ️ src relatif vs absolut
Path relatif: src="gambar/foto.jpg" — lokasi file relatif terhadap file HTML. Dipakai untuk file yang ada di komputer/server kamu sendiri.
Path absolut: src="https://website.com/foto.jpg" — URL lengkap. Dipakai untuk file dari internet.

06

Tabel HTML

Tabel digunakan untuk menampilkan data terstruktur dalam baris dan kolom. Struktur dasar tabel terdiri dari beberapa tag yang bekerja sama:

  • <table> — wadah utama tabel
  • <caption> — judul tabel
  • <tr> — table row (baris)
  • <th> — table header (kolom judul, tebal dan tengah)
  • <td> — table data (sel data biasa)
💡 Analogi
Tabel HTML seperti spreadsheet Excel. <tr> adalah baris (row 1, 2, 3...), <td> adalah sel dalam baris itu. Yang membedakan: <th> adalah header kolom (seperti baris pertama yang ditebalkan di Excel).

⚠️ Common Mistake — Hitung sel dengan cermat!
Saat menggunakan colspan atau rowspan, sel yang "dikuasai" tidak boleh ditambahkan lagi di baris berikutnya. Jika kamu memakai colspan="3" di satu sel, baris itu hanya perlu 1 <td> untuk 3 kolom. Baris berikutnya tetap punya 3 <td> normal.

Form & Jenis-Jenis Input

Form adalah cara utama pengguna berinteraksi dan mengirimkan data di website. Tag <form> adalah wadah untuk semua elemen input. Tag <input> adalah elemen paling serbaguna — tampilan dan fungsinya berubah sesuai nilai atribut type-nya.

💡 Analogi
<form> seperti amplop surat. Semua isian (nama, alamat, pesan) dimasukkan ke dalam amplop, lalu amplop dikirim ke tujuan yang ditentukan oleh atribut action. Tag <input> adalah kolom isian di dalam formulir itu.

Atribut Input Penting

Setiap <input> bisa memiliki atribut tambahan yang mengontrol perilaku dan validasinya. Ini membuat form lebih user-friendly dan aman tanpa perlu JavaScript.

AtributFungsiContoh

requiredWajib diisi — form tidak bisa submit jika kosong<input type="text" required>
disabledInput tidak bisa diubah dan tidak terkirim<input type="text" disabled>
autofocusInput ini otomatis aktif saat halaman dimuat<input type="text" autofocus>
autocompleteMengontrol saran otomatis browser (on/off)autocomplete="off"
valueNilai awal yang sudah terisivalue="Rp. 150.000"
min / maxBatas minimum/maksimum (untuk type number/date)min="10" max="100"
acceptFilter jenis file untuk input type fileaccept="image/*"

✅ Tips — placeholder vs value
placeholder: teks petunjuk abu-abu, hilang saat user mengetik, TIDAK dikirim ke server.

Gunakan placeholder untuk contoh format input, dan value untuk nilai default yang sebenarnya.
09

Struktur Form: fieldset, label & details

Form yang baik bukan hanya soal input — tapi juga soal organisasi dan aksesibilitas. Ada beberapa tag yang membantu mengelompokkan dan memperjelas elemen form:

  • <fieldset> + <legend> — mengelompokkan input terkait dengan kotak dan judul
  • <label> — menghubungkan teks keterangan dengan input secara semantik
  • <details> + <summary> — konten yang bisa dibuka/ditutup (collapsible)
💡 Analogi
<fieldset> seperti kotak formulir fisik yang punya judul ("Data Pribadi", "Data Orang Tua"). Semua kolom isian terkait dikelompokkan di dalamnya. <label> seperti tulisan di samping kotak isian — saat kamu klik tulisan itu, kursor langsung masuk ke kotak isian tersebut.
  Demo 9 — fieldset, label & details

✅ Kenapa label penting?
Dengan <label for="id"> yang dihubungkan ke <input id="id">, pengguna bisa mengklik teks labelnya dan kursor langsung masuk ke kolom isian. Ini sangat membantu di mobile dan bagi pengguna screen reader (aksesibilitas).

Mini Project: Formulir Pendaftaran SMK

Saatnya menggabungkan semua yang telah dipelajari! Kamu akan membuat formulir pendaftaran siswa SMK yang lengkap — mencakup <fieldset> untuk pengelompokan data, berbagai jenis <input>, <select>, <radio>, dan validasi dengan required.


Form ini harus memiliki bagian:

  1. Data Calon Siswa (nama, tempat lahir, jenis kelamin, alamat, tanggal lahir, NIK, NISN, agama)
  2. Data Orang Tua / Wali (nama, pekerjaan, penghasilan)
  3. Pilihan Jurusan
  4. Pernyataan persetujuan (<details> + checkbox)
  5. Tombol Kirim
💡 Strategi Pengerjaan
Kerjakan section per section. Buat satu <fieldset>, isi dengan input-input di dalamnya, uji di browser, baru lanjut ke fieldset berikutnya. Jangan mencoba menulis semua sekaligus!
  Kode Mini Project — Formulir Pendaftaran SMK 
<h2>Formulir Pendaftaran SMK Sekolahku</h2>

<form action="">

  <!-- ── BAGIAN 1: DATA CALON SISWA ── -->
  <fieldset>
    <legend>Data Calon Siswa</legend>

    <p>
      <label for="nama">Nama Lengkap:</label>
      <input type="text" id="nama"
             placeholder="Nama Calon Siswa"
             required>
    </p>

    <p>
      <label>Jenis Kelamin:</label>
      <input type="radio" name="jk"> Laki-laki
      <input type="radio" name="jk"> Perempuan
    </p>

    <p>
      <label for="tgl">Tanggal Lahir:</label>
      <input type="date" id="tgl">
    </p>

    <p>
      <label for="agama">Agama:</label>
      <select id="agama">
        <option>Islam</option>
        <option>Kristen</option>
        <option>Katolik</option>
        <option>Hindu</option>
        <option>Buddha</option>
      </select>
    </p>
  </fieldset>

  <!-- ── BAGIAN 2: DATA ORANG TUA ── -->
  <fieldset>
    <legend>Data Orang Tua</legend>

    <p>
      <label for="ayah">Nama Ayah:</label>
      <input type="text" id="ayah">
    </p>

    <p>Penghasilan Ayah:
      <input type="radio"
             name="gaji_ayah"> < 3 juta
      <input type="radio"
             name="gaji_ayah"> 3–5 juta
      <input type="radio"
             name="gaji_ayah"> > 5 juta
    </p>
  </fieldset>

  <!-- ── BAGIAN 3: JURUSAN ── -->
  <p>
    <label for="jurusan">Pilih Jurusan:</label>
    <select id="jurusan">
      <option>Teknik Komputer</option>
      <option>Teknik Informatika</option>
      <option>Teknik Kimia</option>
    </select>
  </p>

  <!-- ── BAGIAN 4: PERNYATAAN ── -->
  <details>
    <summary>Pernyataan & Persetujuan</summary>
    <p>Saya menyatakan data yang diisi adalah benar.</p>
    <p>
      <input type="checkbox" required>
      Saya setuju dengan persyaratan di atas.
    </p>
  </details>

  <br>
  <button type="submit">
    Kirim Data Pendaftaran
  </button>

</form>
✅ Tantangan Tambahan
Setelah berhasil membuat form dasar, coba tambahkan:
  1. Bagian "Data Ibu" mirip seperti "Data Ayah"
  2. Field NISN dan NIK dengan placeholder format yang benar
  3. Upload foto dengan <input type="file" accept="image/*">
  4. Beri required pada field-field yang wajib diisi
🤔 Refleksi Akhir Modul: Kamu sudah belajar dari tag teks sederhana sampai form yang kompleks. Coba buka browser, ketik F12, dan klik "Inspect" pada sebuah website favoritmu — kamu sekarang sudah bisa membaca dan memahami kode HTML yang ada di sana!
